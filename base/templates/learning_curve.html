- UseEffect = React Hook that tells React DO SOME CODE WHEN:

1- This component re-renders.

2- This component mounts.

3- The state of a value.

1. useEffect(() = ()) Runs after every re-render.

2. useEffect(() = (), []) Runs only on mount.

3. useEffect(( => (J, [value]) // Runs on mount + when value changes.

USES:

1- Event Listeners.

2- DOM manipulation.

3- Subscriptions (real-time updates).

4- Fetching Data from an API.

5- Clean up when a component unmounts.

- useRef()= use Reference" Does not cause re-renders when its value changes. When you want a component to "remember" some information, but you don't want that information to trigger new renders.

1- Accessing/Interacting with DOM elements.

2- Handling Focus, Animations, and Transitions.

In summary:
- useRef is for storing values that don't directly affect the UI and don't trigger re-renders when updated. useState is for managing stateful values that directly affect the UI and trigger re-renders when updated.

- JavaScript BOM is the interaction between JavaScript and the browser.

- JavaScript DOM is the interaction between JavaScript and HTML.

- A component is like a section from many sections that consists of a site; it's reusable and can be a function or a class.

- Features of React:

  • JSX (JavaScript XML): allows us to write HTML in JS code,

  • One-way binding: This means that a variable can change in JS and HTML without reloading the page,

  • Virtual DOM,

  • Simple and rapid, SEO-friendly, size compression of the code in the build phase.

- React concepts:

  • Components.

  • State.

  • Lifecycle.

  • Props.

  • JSX.

- We can use React with CDN and CLI.

- Props:

It's an object that gives us the ability to pass data from the father component to the child component. It's public and Immutable.

- State:

It's private and mutable but can pass state through components using props.

- React events:

  • onChange,

  • onClick.


  • - Array iteration using map().

- We should use key=index in map to avoid confusion.

- Styling with ReactJS (CSS): Inline, module, external, also we can use the CSS-in-JS libraries: (import styled from 'styled-components';) and Emotion (import { css } from '@emotion/react';).


- React Router:

Wrapping the entire application <BrowserRouter> provides the routing functionality for React applications using HTML5 history API.

Responsible for all routing in React. We can use <Switch> to avoid any router confusion. Also, exact to get to the exact URL. history.push to get to the destined URL.

<Route> component is used to render a component based on the current URL.

<Link> component is used to create links that navigate between different pages.


- Lifecycle of a component:

  • Initialization: constructor - use it for variable declaration, states.

  • Mounting: componentDidMount - initialization of the component in the DOM. render - render the resulting HTML or JS to us.

  • Updating: shouldComponentUpdate -> componentWillUpdate -> componentDidUpdate - when an update happens to the component.

  • Unmounting: componentWillUnmount - unlike componentWillMount.

- Axios:

Responsible for the connection between the frontend and backend.

- Async function:

await is used when we want something to finish before going to the next line.

- Redux:

Suitable for more complex projects, used to transfer data between components, better than props, and we can avoid component drilling with it. Redux is centralized data and synchronization of the data shared between components (view), modified by the reducers. const store = createStore().

- React Hooks:

Since we can't use state and lifecycle components in function components, hooks let us do that. Their functions and ways to interact with component lifecycle and state management.

  • useState: Enables the function component to have a state, return stateful value and update it.

  • useEffect: Replaces componentDidMount, componentDidUpdate, componentWillUnmount. It gives you the ability to do a side effect function using a happened effect. We can use a list dependency to let React know that we want useEffect to be used as necessary or let it be blank so it's used only once.

  • useRef: Able to take an object as a reference. The current property contains information related to the referred object.

- Laravel Documentation:

- dd stands for "dump and die" in Laravel. It is a debugging function that prints the variables and halts the execution of the script.

- ddd provides a more detailed and interactive debugging experience.

Laravel Project Structure:

- 'App'folder contains the core application logic: controllers, models, middleware…

- 'Bootstrap'folder contains the app.php file that sets up the application and loads the necessary dependencies.

- Middleware: They intercept and modify requests before they reach the application.

we can group middleware and apply it to the routes, example:


            Route::middleware(['auth', 'admin'])->group(function () {
                // Routes requiring both authentication and admin privileges
            });

- Views are responsible for rendering the HTML that is sent to the browser. They are typically created using the Blade templating engine.

Routing:

api.php is a file where you define routes specifically for API endpoints, allowing for a separate route file dedicated to API routes.

web.php is a file where you define web routes, specifying how the application responds to HTTP requests.


            Route::get('/', [HomeController::class, 'index']);

=> The root URL (/) is mapped to the index method of the HomeController.


            Route::get('/about', function () {
                return view('about');
            });

=> Returns a view named "about".

- Blade: Laravel's templating engine for creating web pages with PHP integration in HTML.

- Artisan is command-line interface (CLI) tool of Laravel.

- php artisan make:model: used to generate a new model class. (same cmd for controller, middleware, or migration…).

ORM:

- Eloquent is Laravel's ORM (Object-Relational Mapping) that simplifies database interactions by allowing you to work with databases using an expressive syntax in PHP.


Examples:


            // Retrieve all users
            $users = App\Models\User::all();
            
            // Retrieve a single user by ID
            $user = App\Models\User::find(1);
            
            // Retrieve users with a specific condition
            $adminUsers = App\Models\User::where('role', 'admin')->get();
            
            // Create a new user
            $newUser = App\Models\User::create([
                'name' => 'John Doe',
                'email' => 'john@example.com',
                'password' => bcrypt('password'),
            ]);
            
            // Update an existing user
            $user->update(['name' => 'Jane Doe']);
            
            // Delete a user
            $user->delete();

- Supports relationships between models like: hasOne, hasMany, belongsTo, belongsToMany.

- Eloquent Observers allow one to watch for certain events on a model and perform actions in response to those events.


            // User model
            class User extends Model {
                // ...
            }
            
            // UserObserver
            class UserObserver {
                public function created(User $user) {
                    // Action to perform when a new user is created
                }
            
                public function updated(User $user) {
                    // Action to perform when a user is updated
                }
            }

=> PS: we must register the observer In the service provider 'App\Providers\EventServiceProvider'.

- Scope filtering is a way to apply filters to your queries within Eloquent models, Example:


            // Inside the Post model

            class Post extends Model
            {
                public function scopePublished($query)
                {
                 return $query->where('status', 'published');
                }
            }

=> Now we use this scope in our controller to retrieve published books.


            $publishedPosts = Post::published()->get();

Form Validation:

- Laravel simplifies form validation with a built-in validation rules, example:


            class UserController extends Controller
            {
                public function store(Request $request)
                {
                    // Validate the incoming request data

Authentication:

- Generate the authentication scaffolding: php artisan make:auth.

- Run migration: php artisan migrate.

- Update User Model by adding ‘Authenticatable’ trait to it, to provide the necessary methods for user authentication.

- We can Display Authentication Status in Views:


            {% verbatim %}
        @auth
            {{-- The user is authenticated --}}
        @else
            {{-- The user is not authenticated --}}
        @endauth
        {% endverbatim %}

- We can also access the authenticated user:


        @auth
        {% verbatim %} Welcome, {{ auth()->user()->name }}{% endverbatim %}
        @endauth
